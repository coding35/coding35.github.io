The Singleton Pattern ensures a class has only one instance, and provides a global point of access to it. This pattern is used when we want to ensure that only one object of a particular class is instantiated. There are several ways to implement the Singleton pattern.

  • Lazy Singleton Initialization
  • Eager Singleton Initialization
  • Singleton with Enum

Lazy Singleton initialization is the most common way and default way to implement the Singleton. The Singleton object is created only when it is needed. This singleton is not thread-safe. If multiple threads try to access the singleton at the same time, they may end up creating multiple instances of the singleton. This can be fixed by using code to synchronize the getInstance() method. However, this can cause performance issues.

Eager Singleton initialization is the simplest way to implement the Singleton. The Singleton object is created when the class is loaded into the memory. This singleton is thread-safe. However, this can cause performance issues if the singleton is not used frequently.

Singleton with Enum is the best way to implement the Singleton. The Singleton object is created when the class is loaded into the memory. This singleton is thread-safe. This is the best way to implement the Singleton because it is simple, provides serialization for free, and provides protection against multiple instantiation.

Notes

  • A single, private, parameterless constructor.
  • Should be marked as sealed.
  • A private static field holds the only reference to the instance in the singleton class it self.
  • The application accesses the singleton using a public static method.
  • In C# Static Constructors run only once per appdomain which makes it good for Singleton Use. This avoids having beforefieldinit applied which makes our singleton behavior lazier. This also removes the need for locks and improves performance. See Nested Class example
  • Look to use Lazy<T> for thread safety and performance.
