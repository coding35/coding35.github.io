The Anti-Corruption layer Pattern first proposed by Eric Evans in Domain-Driven Design promotes the idea that an application's design not be limited by dependencies outside its domain. For example, a domain model should not be limited by the data structures of a legacy database or the APIs of a legacy system.

In short this pattern is used to isolate an application domain from external dependencies by providing a translation layer between itself and dependant subsystems. This layer ensures that the calling application is not beholden to data structures, obsolete technologies and APIs outside of its control. This pattern can be particularly useful in legacy migrations or when two systems with different data models or semantics need to communicate.

Considerations

  • Added latency
  • Management and scaling
  • Possible Partitioning
