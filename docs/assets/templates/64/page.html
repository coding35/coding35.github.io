Pattern Description
Adapter Pattern Changes the interface of one or more classes.
Class Adapter Pattern Uses multiple inheritance to adapt one interface to another.
Command Pattern Decouple the sender of the request from the object that executes it.
Composite Pattern Clients treat collections of objects and individual objects uniformly.
Decorator Pattern Allows for the dynamic wrapping of objects in order to modify their. behavior
Facade Pattern Simplifies the interface of a group of classes
Factory Method Pattern Creates an instance of several derived classes.
Iterator Pattern Provides a way to traverse a collection of objects without exposing the collection's implementation.
Observer Pattern Allows a group of objects to be notified when some state changes.
Proxy Pattern Wrap an object to control access to it.
State Pattern Encapsulates state-based behavior and uses delegation behavior to the current state.
Strategy Pattern Encapsulates interchangeable behaviors and uses delegation to decide. which one to use
Template Method Pattern Create a template for an algorithm and let subclasses redefine certain steps.
