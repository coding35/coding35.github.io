The command pattern is a behavioral design pattern that is useful when it is necessary to separate the request from the execution logic of the request. It is often used to log and audit changes in a system.

The command pattern encapsulates a request as an object to decouple the sender of the request from the object that executes the request.

  1. Client - Creates a command object and sets its receiver
  2. Command - Encapsulates a request as an object
  3. Invoker - Asks the command to carry out the request
  4. Receiver - Knows how to perform the operations associated with carrying out the request
