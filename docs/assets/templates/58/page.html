The template pattern allows you to define the skeleton of an algorithm in a method, deferring some steps to subclasses for implementation. This pattern gives allows for code reuse while still allowing subclasses to define some of the steps of the algorithm through the use of the concept of a hook. A hook is simply a method that is declared in the abstract class, but only given an empty or default implementation. This method can be overridden by subclasses, but does not have to be.

This pattern shows up so often because it's a great design tool for creating frameworks, where the framework controls how something gets done, but leaves you (the person using the framework) to specify your own details about what is actually happening at each step of the framework's algorithm.
