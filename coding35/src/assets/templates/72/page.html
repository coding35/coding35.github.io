The proxy pattern is a structural design pattern that lets you substitute a placeholder for another object to control access (local or remote), that is expensive to create, or is in need of securing. It can be used when you want to add some additional behaviors to an object of some existing class without changing the client code but there may be a better strategy to adding additional behavior.
Basically, the proxy intercepts all calls to the real object and does something like logging, caching, checking authentication/authorization making a request to a remote service, returning a placeholder value, etc.

Remote Proxy: A proxy that acts like a local representative for an object that lives in a different/remote address space.

Virtual Proxy: A proxy that acts as a placeholder for an expensive to create object and will often times defer the creation of the object until needed. One use case may be to display a loading spinner while the object is being created and the request is being processed.

Protection Proxy: A proxy that controls access to an object based on permissions.
