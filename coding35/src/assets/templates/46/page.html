The observer pattern is a very common design pattern where by a one-to-many relationship is defined between a set of objects. When the state of one object changes, all of its dependents are notified using a common interface. Any object can be an observer as long as it implements the observer interface. Since the subject knows nothing about the observers the relationship between the object is loosely coupled. This allows for the addition of new observers without the need to change the subject.

This pattern is often used in event handling system such as graphical user interfaces. By separating the presentational aspect from the underlying application logic, the application can be more easily extended to support new types of events.
