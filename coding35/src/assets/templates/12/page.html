In object-orientated programming, there are a handful of class types to build objects.

Concrete Class

A class with defined properties and methods that can be instantiated. It can be extented and inherited by other classes.

Usage
  • Reusability by means of inheritance to subclasses.
  • Implementation of an interface.

Abstract Class

A class that cannot be instantiated but provides an interface or blueprint for other classes. Abstract classes can contain abstract methods which are declared but not defined. These abstract methods must be implemented by the subclass.

Usage
  • Requires defined methods that must be implemented by all subclasses.
  • To force a specific pattern or interface for objects in a system.
  • Provide both default implementation of methods that can be overridden and defined methods that require implementation by subclasses.
  • Ensure that code is structured correctly, and all subclasses conform to a predefined structure.

Final Class

A class that does not allow for inheritance subclassing. Classes to be used without modification.

Usage
  • Security
  • Tamper proof code.
  • Optimization
  • Prevents inheritance.

Static Class

A class that cannot be instantiated and contains only static properties and members and generally used as utility functions or managing global state.

Usage
  • Reusable utility or helper methods.
  • Useful to hold constants or state.

Inner Class

A class that is defined within the confines of another class that can access the outer classes members.

Usage
  • Encapsulate related functionality within a class.
  • Event handling such as a listener or callback.
  • Access to private members of the outer class

Anonymous Class

A dynamic class that is defined at runtime and has no name. This type of class is generally used for creating objects with basic functionality.

Usage
  • Event handling such as a listener or callback.
  • As part of an inner class.
  • Useful in testing.
  • Temporary implementation of an interface or abstract class.
