Corporate proxies are a common way to control internet access. However, they can cause problems when trying to access external services. For example, when trying to access GitHub Copilot, the following error message is displayed:

[Error] An error occurred for 'RunOperation:RunOperation': StreamJsonRpc.RemoteInvocationException: Request signInInitiate failed with message: self signed certificate in certificate chain at StreamJsonRpc.JsonRpc.d__1451.MoveNext() --- End of stack trace from previous location where exception was thrown --- at System.Runtime.ExceptionServices.ExceptionDispatchInfo.Throw() at System.Runtime.CompilerServices.TaskAwaiter.HandleNonSuccessAndDebuggerNotification(Task task) at GitHub.Copilot.CopilotClient.d__201.MoveNext()

Steps

  1. Navigate to Github.com.
  2. Inspect the SSL certificate.
  3. View the certificate hierarchy and export the top level certificate listed.
  4. Open command prompt as admin.
  5. Excecute setx NODE_EXTRA_CA_CERTS "path\to\cert.pem"
  6. Restart Visual Studio.
