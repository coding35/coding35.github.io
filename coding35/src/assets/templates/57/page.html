The Principle of Least Knowledge (PLK) or Law of Demeter (LOD) is a design guideline useful for developing object-oriented programs. The principle states that for any object, it is important to control the number of classes that it interacts with. This guideline ensures that our classes are loosely coupled thereby making code easier to maintain and extend.

The overall goal of the PLK is to reduce the dependencies between classes and promote encapsulation. An object should only interact with its immediate neighbors and not with objects further away. An object should only communicate with objects through its own interface.

"Only talk to your immediate friends."

An object should only call methods of:

  • Itself
  • Objects passed in as a parameter to the method
  • Any object it creates
  • Any components of the object
  • Method parameters

Following PLK should ensure a level of orthogonality in our code whereby a component is independent of its peers. The more orthogonal a component is, the less it knows about its peers and changes in one part of the code will not affect other parts.

Example

    public class A
    {
        public void DoSomething(B b)
        {
            b.DoSomething();
        }
    }
    
    public class B
    {
        public void DoSomething()
        {
            C c = new C();
            c.DoSomething();
        }
    }
    
    public class C
    {
        public void DoSomething()
        {
            // Perform some action
        }
    }
